Accessibility Assessment

Based on the Web Content Accessibility Guidelines (WCAG), this website success criteria aims:

  • Perceivable

  • 1.1.1. Non-text content

    Ensures that all users, including those using screen readers, can access content conveyed through images, icons, or multimedia by providing text alternatives.

  • 1.3.2 Meaningful Sequence

    Guarantees that content is presented in a logical order, which is crucial for users relying on assistive technologies to understand the flow and meaning of information.

  • Operable

  • 2.4.2 Page Titled

    Provides clear and descriptive titles for each page, aiding users in navigation and orientation, especially those using screen readers or browser tabs.

  • 2.4.4 Link Purpose (In Context)

    Ensures that the purpose of each link is clear from its context, helping users with screen readers or cognitive disabilities to make informed decisions about navigation.

  • Understandable

  • 3.1.5 Reading Level

    Ensures that content is written in a way that is understandable to a wide audience, including those with limited literacy or cognitive disabilities.

  • 3.2.3 Consistent Navigation

    Promotes predictability by keeping navigation mechanisms consistent across pages, reducing confusion and cognitive load.

  • 3.3.1 Error Identification

    Helps users recognize when an error has occurred (e.g., in a form), which is essential for correcting mistakes and completing tasks.

  • 3.3.4 Error Suggestion

    Offers suggestions for fixing errors , which supports users in successfully completing forms or processes.
